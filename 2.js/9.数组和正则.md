正则在非全局(g)模式下，通过match方式，返回的数组第一个值整体匹配的字符串，其他值为通过括号分组匹配到的
```
'adobe cs9'.match(/[a-z]+\d+/);
// ["cs9"]
'adobe cs9'.match(/[a-z]+(\d+)/);
// ["cs9", "9"]
```

### (?=) 非捕获用法多层理解
(?=p)，其中p是一个子模式，即p前面的位置。
要求接下来的字符与p匹配，但不能包括p的那些字符
在本人看来(?=p)就与^一样好理解，就是p前面的那个位置
把位置理解空字符，是对位置非常有效的理解方式
